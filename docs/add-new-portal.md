# Add New Portal Guide

Use this only when adding a brand new role portal (example: `merchant`).
Reference implementation is current `admin` portal.

## Scope

Adding a portal touches:

1. Backend auth + contracts + routes
2. Frontend app entry + Vite config
3. Build/dev orchestration

## Backend Checklist (Rust)

1. `app/configs.toml`
Add new guard config:
`[auth.guards.{portal}]` with `provider`, `ttl_min`, `refresh_ttl_days`.

2. `app/schemas/{portal}.toml`
Define auth model and required enums (`auth = true`, `auth_model = "{portal}"`).

3. `app/permissions.toml`
Add permission catalog entries scoped to the new guard.

4. `migrations/{next}_{portal}_auth.sql`
Create table/indexes for the new auth model.

5. Contracts
Add DTOs under:
- `app/src/contracts/api/v1/{portal}/account.rs`
- `app/src/contracts/api/v1/{portal}/auth.rs`
- `app/src/contracts/datatable/{portal}/...` (if datatable needed)

6. Workflows
Add:
- `app/src/internal/workflows/{portal}.rs`
- `app/src/internal/workflows/{portal}_auth.rs`

7. Middleware auth helper
Extend `app/src/internal/middleware/auth.rs` with `require_{portal}`.

8. API handlers
Add:
- `app/src/internal/api/v1/{portal}/account.rs`
- `app/src/internal/api/v1/{portal}/auth.rs`

9. Route wiring
Mount new routes in `app/src/internal/api/v1/mod.rs`.

10. Datatable wiring (if used)
Add portal executor under `app/src/internal/datatables/v1/{portal}/...` and register in `app/src/internal/api/state.rs`.

11. Seed data
Add portal bootstrap seeder under `app/src/seeds/`.

12. i18n
Add keys to all `i18n/*.json`.

13. Module declarations
Wire `mod` / `pub mod` in all affected `mod.rs`.

## Rust API SPA Serving

Update `app/src/internal/api/mod.rs` and mount the new portal SPA **before** user catch-all.

Pattern:

```rust
// /{portal}/* -> public/{portal}/index.html
let portal_public = public_path.join("{portal}");
let portal_index = portal_public.join("index.html");
if portal_public.is_dir() && portal_index.is_file() {
    router = router.nest_service(
        "/{portal}",
        ServeDir::new(&portal_public).fallback(ServeFile::new(&portal_index)),
    );
} else {
    router = router
        .route("/{portal}", axum_get(portal_dev))
        .route("/{portal}/{*path}", axum_get(portal_dev));
}
```

Add `{portal}_dev()` handler by copying `admin_dev` and changing Vite port + entry script.

## Frontend Checklist

1. `frontend/vite.config.{portal}.ts`
Set:
- `base: "/{portal}/"`
- unique `server.port`
- `build.outDir: "../public/{portal}"`

2. `frontend/{portal}.html`
Entry HTML with script to `/src/{portal}/main.tsx`.

3. `frontend/src/{portal}/main.tsx`
Use `<BrowserRouter basename="/{portal}">`.

4. `frontend/src/{portal}/App.tsx`
Add route tree and `ProtectedRoute`.

5. `frontend/src/{portal}/app.css`
Define portal theme tokens and `rf-*` classes.

6. `frontend/src/{portal}/api.ts`
Create Axios client via `createApiClient`.

7. `frontend/src/{portal}/stores/auth.ts`
Create store via `createAuthStore` with `/api/v1/{portal}/auth/*`.

8. `frontend/src/{portal}/types/`
Generated by `make gen-types`; do not hand-edit.

9. `frontend/package.json`
Add `dev:{portal}` and `build:{portal}` scripts.

10. Build order
Ensure `build:{portal}` runs before `build:user` so nested portal outputs are preserved.

## Makefile Checklist

1. Add `dev-{portal}` target.
2. Add `npm --prefix frontend run dev:{portal}` to the `dev` aggregate target.

## Validation

1. `cargo check --workspace`
2. `make gen-types`
3. `make build-frontend`
4. `./console route list`
5. Verify:
- `/{portal}/login` loads in dev
- protected routes redirect/refresh correctly
- API auth + me + refresh endpoints work
